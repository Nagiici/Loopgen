Quick start & commands

loopgen is a local-first CLI. Install nothing — run it with npx. Full docs live in the GitHub README (also bilingual EN/ZH).

1. Generate a loop, then run it

loopgen init / apply scans your repo and writes inspectable bounded-loop configs to .loopgen/loopgen.loop.yaml. loopgen run then executes one — running your real verification and leaving proof.

npx loopgen init                 # local wizard, nothing written
npx loopgen run test-repair      # run + verify a loop; exits 0/1 for CI

loopgen run diffs the working tree, runs the loop's verification.commands (gating on exit codes), checks forbiddenPaths, and writes a hash-chained .loopgen/audit.jsonl + a markdown proof report in .loopgen/reports/.

FlagMeaning
--dry-runRun the checks, write nothing.
--base <ref>Git ref to diff against (default HEAD).
--jsonPrint the run result as JSON.

2. Driven mode — loopgen runs the loop

Drive a local model through the loop; guardrails are enforced at apply time (forbidden-path writes blocked before they land, command allowlist, iteration/timeout caps).

npx loopgen run test-repair --mode driven \
  --adapter ollama --ollama-model qwen2.5-coder
FlagMeaning
--adapterollama | openai-compatible
--ollama-model / --openai-compatible-modelModel name (or set it in .loopgen/adapters/<id>/config.json).
--max-iterations <n>Override the loop's iteration cap.
--allow-dirtyRun with a dirty working tree.

3. Governance — loopgen audit

Turn per-repo ledgers into team evidence and a CI gate. Local-first, no server.

npx loopgen audit verify                 # prove the hash chain is intact
npx loopgen audit summary                # one repo: pass rate, by loop, violations
npx loopgen audit aggregate ./repos --html gov.html --report gov.md
npx loopgen audit check --require test-repair --require-no-violations --require-chain

audit check exits non-zero if a required loop has no passing run, a run touched forbidden paths, or the chain is broken — wire it into CI to block merges on missing or tampered proof.

GitHub Action

- uses: Nagiici/Loopgen/.github/actions/audit-check@v0.6.2
  with:
    require: test-repair,ci-failure-repair
    require-no-violations: "true"
    require-chain: "true"

Honest scope

Referee mode is detection (post-hoc git diff). Driven mode enforces at apply time but is not a sandbox — the model still proposes; loopgen bounds, confines, verifies, and proves. No telemetry; only your local models are called; API keys are referenced by env-var name only.
